# 🀄 德扑道场 (Poker Dojo)

> 好友开房、切磋成长，把每一手牌沉淀成 AI 的训练数据。

德扑道场是一款面向好友对战的**服务端权威**在线德州扑克。所有游戏逻辑(洗牌、发牌、牌力评估、下注、结算)都在服务器执行,客户端只负责展示与传送操作。内置大厅、SNG 锦标赛、现金"训练赛"、完整多人引擎(真边池、多次发牌)、牌谱、生涯统计、聊天/表情/语音,以及 Capacitor 安卓薄壳。

> **健康游戏提示:** 本项目是供好友**练习娱乐的虚拟金币游戏**,**不是赌博**——无真钱下注、不可提现、无奖金。绿色竞技,远离赌博。

## ✨ 功能特性

- **服务端权威架构** —— 底牌用 `io.to(socketId)` 私发,广播只含公共信息,所有操作服务端校验;玩家爬自己的流量也看不到对手底牌,无法作弊。
- **可验证公平洗牌** —— 每手重建整副牌,Fisher–Yates + **密码学安全随机**(`crypto.randomInt`),无偏且不可预测;蒙特卡洛自检各牌型频率与理论值吻合。
- **两种房型**
  - **SNG(单桌锦标赛)**:盲注按时间递增、淘汰制、奖池抽水。
  - **现金 / "训练赛"(2–9 人)**:固定盲注,金币↔筹码带入/兑出(含价差抽水),训练时长 + 加时。
- **完整多人引擎** —— 正确的行动顺序(UTG 起手)、按钮轮转、**真边池**(相邻同资格档位合并 + 未跟注退还)。
- **多次发牌(run it N times)** —— 两人 all-in 时,落后方选发几次(1–5)、领先方同意,底池均分 N 份,在桌面上逐街依次发出。
- **牌谱 = 数据资产** —— 每手完整记录(各街动作、思考时间、底牌、公共牌、结果、时间戳),按玩家、按模式归档为追加式 JSONL,同时作为 **AI 训练数据**。
- **生涯统计** —— VPIP / PFR / 3-bet / C-bet / AF / WTSD / 盈亏 + 盈亏曲线,从牌谱聚合。
- **牌桌体验** —— 环形座位、头像、呼吸行动框 + 环形倒计时、四色扑克、下注滑条、预操作、全押实时胜率%、筹码/底池动画、手机振动。
- **社交** —— 桌内聊天 + 快捷短语、点头像发表情、按住说话语音气泡。
- **账号与安全** —— 用户名/邮箱注册(邮件 6 位验证码),JWT 鉴权(每台服务器独立随机私章),生产环境 TLS/WSS。
- **安卓** —— **Capacitor** 薄壳指向线上站点;游戏更新只需部署服务器,无需重新发包。

## 🧱 架构

```
浏览器 / 安卓 (薄客户端)  ──socket.io──►  Node 服务器 (权威)
   只负责展示 + 输入                        洗牌 · 发牌 · 判牌
                                            下注 · 边池 · 结算
                                            牌谱落库 (JSONL)
```

- **服务端权威**:规则全在服务器;客户端拿不到对手底牌。
- **牌力评估**:Cactus Kev + Paul Senzee 完美哈希。每张牌编码为 32 位整数;7 张枚举 C(7,5);分数 1(最强)…7462(最弱)。JS 版用 `>>> 0` 保持无符号 32 位运算。
- **存储**:内存游戏状态 + JSON 文件(`data.json` 用户库、`hands.jsonl` 牌谱)。`database.js` 对外接口稳定,后续可平滑迁移 SQLite/Postgres 而不动调用方。

### 目录结构

| 路径 | 说明 |
|------|------|
| `PokerServer/` | Node.js 游戏服务器(Express + Socket.IO)与单页客户端(`index.html`) |
| `PokerLogic/`  | 牌力评估算法的 C# 原型(算法来源;JS 端为权威实现) |
| `mobile/`      | Capacitor 安卓薄壳(`capacitor.config.json` 指向线上地址) |
| `docs/`        | 设计文档 |

**技术栈:** Node.js · Express · Socket.IO · 原生 HTML/CSS/JS 客户端 · JWT · bcrypt · nodemailer · JSON 文件存储。

## 🚀 本地快速开始

```bash
cd PokerServer
npm install
LOCAL_DEV=1 PORT=3000 node server.js
# 打开 http://127.0.0.1:3000
```

`LOCAL_DEV=1` 会绑定 localhost 并预置两个测试账号(`test`/`test`、`test2`/`test2`),可直接登录。未配置 SMTP 时,邮件验证码会回退打印到服务器日志。

> 生产部署(服务器配置、TLS、备份等)刻意不写进本 README。
