# PokerGame 项目说明（产品名：德扑道场 Poker Dojo）

## 项目目标
开发一款支持好友在线开房同玩的德州扑克游戏。**产品名：德扑道场（Poker Dojo）** —— 好友开房、切磋成长，牌谱沉淀为 AI 训练数据。

## 目标平台
Android / iOS / PC

## ⭐ 数据资产与可迁移性（最重要原则）
**数据是这个项目最宝贵的资产，比服务器本身更重要。** 底层代码可随时重新部署，但用户数据与对局数据必须能跟着走。

- **服务器是临时的**：现在跑在阿里云香港，到期后会迁移到新服务器。迁移时**代码不变，但所有数据必须一起搬过去**。
- **当前数据存储**：服务器内存 + `PokerServer/data.json`（用户库：账号/金币/头像/isAdmin）。该文件**被 .gitignore 排除、deploy 脚本也不覆盖它**（tar 排除 data.json），所以发版不会丢数据——但也**不会自动备份**。
- **迁移/备份必做**：换服务器时，从旧机 `scp` 下 `data.json`（及将来的牌谱库）到新机，否则用户全丢。建议定期备份 data.json。
- **牌谱（hand history）是核心数据资产**（后期务必实现，见下）：
  - 完整记录每一手：各街动作、**思考时间**、底牌、公共牌、下注额、底池、结果、对手、**时间戳**
  - **按玩家分别归档**：每个真人玩家的历史行为是极有价值的 **AI 训练数据**——尤其针对特定玩家形成的个性化数据集，可训练"模仿某人打法"的 AI
  - **再按游戏模式细分归档**：同一玩家在 **SNG 锦标赛升盲** 与 **现金桌** 下的策略不同（短码/ICM vs 深筹码），数据集要按 `(玩家 × 模式)` 切分，别混在一起
  - **保留时间先后顺序**：人会学习、策略会随时间演变，所以每手的**先后顺序/时间戳是有效信号**，不是噪声——训练/分析时要保留时序，可建模"某人打法的演变"，也意味着旧数据权重可能要衰减
  - 同一份数据同时驱动：**个人数据统计**（VPIP/PFR/3-bet…）、**牌谱回顾**、**AI 对战**三大功能
- **演进方向**：data.json 适合当前小规模；用户/牌谱量大后迁移到 SQLite/PostgreSQL（database.js 对外接口不变，便于替换）。牌谱建议从一开始就结构化落库，别只存内存。

## 架构概览

### 服务端权威架构
所有游戏逻辑（发牌、判牌、押注结算）均在服务器执行，客户端只负责展示和传送玩家操作。

### 目录结构
- `PokerLogic/` — C# 核心算法原型（Cactus Kev 牌力评估）
- `PokerServer/` — Node.js 游戏服务器（Express + Socket.IO）

### 牌力评估算法
使用 **Cactus Kev + Paul Senzee 完美哈希**方案：
- 每张牌编码为一个 32 位整数
- 5 张牌评估走 Flushes / Unique5 / PerfectHash 三条路径
- 7 张牌枚举全部 21 种 C(7,5) 组合取最低分
- 分数范围 1（最强同花顺）～ 7462（最弱高牌），**分数越小牌越强**
- JS 版需用 `>>> 0` 保持无符号 32 位运算

### 安全（防作弊 / 隐私）
- **底牌不外泄**：`hole_cards`/`my_hand` 用 `io.to(socketId)` **私发**；广播的 `game_state` 只含公共信息（公共牌/筹码/下注/弃牌），**无任何底牌字段**；`showdown_reveal` 只揭示未弃牌者。→ 玩家爬自己的流量看不到对手底牌（服务器权威架构的核心优势）
- **行动校验**：`player_action` 校验"是否轮到你"+ 下注额服务端 clamp，改客户端无法越权/非法下注
- **JWT 私章已加固**：`JWT_SECRET` 不再用写死默认值（公开仓库可见=可伪造令牌）；改为 env > 本地 `secret.key`（首启自动生成 96 位强随机、落盘复用；**不进 git、deploy 不覆盖**，每台服务器唯一）。换私章后旧登录失效需重登一次（账号金币不丢）
- **仍待办**：①**TLS/HTTPS**（现 http/ws 明文，同网络可嗅探受害者自己的底牌+token；需域名+证书+反代 WSS）②串通 collusion 技术无解（人因，忽略）
  - ⚠️ **生产(香港)需重新 deploy.sh 才会生效 JWT 加固**（部署后会自动生成 secret.key，全体重登一次）

### 发牌随机性（重要：朋友局公平性底线）
- **每手牌都重新 `reset()`+`shuffle()`**（server.js startHand）：Fisher-Yates 无偏洗牌
- **随机源已升级为 `crypto`（CSPRNG）**：`PokerLogic.js` shuffle 用 `crypto.randomInt(n+1)`（无模偏均匀整数），不可预测（看遍历史也推不出后续），替代了原 `Math.random()`（统计公平但理论可预测）
- 每手生成 `deck.lastShuffleId`（crypto 随机 4 字节）并 `console.log('[deal] ... shuffleId=')` 审计，确认确有重洗
- 已自检：两副不同、首张=黑桃A 频率 ≈1/52、各位置分布均匀

## 🔴 行动流程严重bug修复 + 房主功能 + 9max布局（2026-07-24，本地验证通过；测试服5090当时SSH超时未部署）
- **背景**：用户实战反馈多个"卡住牌局"的严重bug。共修 6 个行动流程bug（前 4 个见上一批 commit f985370，后 2 个本批新增），均围绕 `actionOnIdx`/行动推进。
- **上批已修(f985370)**：①弃牌者本手进行中「站起围观」立即 `vacateSeat` splice 打乱 `actionOnIdx`→后面玩家无法行动 → 改为**本手进行中一律延后离座**(`vacateAfter`，`removeBustedPlayers` 处理)；②`findNextActionIdx` 只判 `canAct` 不判"已行动且已跟平"→已跟注者过一圈又被要求过牌 → 加 `needsToAct`；③`reserve_leave` 当前行动者留座→全桌干等到超时 → 弃牌坐出并推进；④加时按钮 400ms 防抖。
- **本批新增(server.js)**：⑤**全押跑马死局**：若所有参与者因盲注/前注**全部全押**(常见于短码 heads-up 双方 all-in from blinds)，`startHand` 里 `actionOnIdx` 会算成 -1 但**没有任何东西触发跑马**→本手永久卡住。修：`startHand` 末尾若 `actionOnIdx<0` 直接 `advanceStage`(进全押亮牌+跑马)；heads-up 时若 SB 已下盲全押则顺延找下一个能行动者(避免超时误弃全押的 SB)。⑥`onActionTimeout` 兜底：`if(!canAct(player)){afterAction;return}`——**绝不把全押/已弃牌者按超时弃牌**(否则剥夺其应得池权)。
- **房主新功能**：**⏸️ 暂停发牌**(`pause_dealing`/`resume_dealing`，owner-only)——暂停后**当前这手打完**不开新局(`game.paused`：`startHand`+`scheduleNextHand` 都 gate)，「继续」立即续局；桌内菜单按钮 `#tmPause` 动态切换文案，中央水印显示「⏸️ 房主已暂停发牌」。**🧍 强制玩家站起**(`force_stand({targetUserId})`，owner-only)——把某玩家移到观战席腾座位(复用 `standUpPlayer` 核心=本手进行中延后离座)，筹码保留至结束结算，TA 可自行回座；入口在**点头像弹层**(房主对其他在座玩家显示「移到观战席」按钮)。
- **9-max布局**(`ringPos`)：M≥6 时顶部座位上提 5% 避开「房间号/盲注」中央水印(修筹码BB数与房间号重合)；「我」的左右两个座位(ringIndex 1 / M-1)上移6%+左右外扩4%，不再和自己挤在一起。
- **点头像「本局盈亏」口径统一**：原取牌谱聚合 `stats.net`(与「当前战绩」面板的 `chips-buyIn` 可能不一致：中途手/补码/房号复用时漂移)→ 改为**同源当前牌桌状态**(`roomStatFor`：在座=chips-buyIn，站起者=vacated.net，已离开=statsHistory.net)，标签改「本房战绩」；手数也改同源。

## 🩹 边池显示 + 未跟注退还（2026-07-24，测试服+香港已上）
- **现象**：多人 all-in 时出现「一长串边池 边池1/2/3/4/5」，最终结算金额正确但层数虚高、看不懂。
- **根因**：`buildSidePots`/`livePots` 对**每个不同投入额**都切一层——**弃牌者的盲注/半路弃牌零头**会切出一个与相邻档位「有资格赢家完全相同」的**多余边池**；**未被跟到的下注**也会残留一个「只有下注者自己有资格」的单人边池。
- **修 1（合并）**：`mergeAdjacentPots` 把相邻「有资格玩家集合完全相同」的档位并成一个池（结算等价，只是不再显示成一串）；`livePots` 同样按 userId 集合合并。
- **修 2（退还未跟注）**：`returnUncalledBets`——最高投入者超过「第二高投入」的部分无人能跟 → 退回给他（`collectBetsToPot` 后调用、幂等；在 all-in 亮牌 + `doShowdown` 各调一次）。**关键**：修正 **run-it 不等筹码结算**——之前 A 全押 5000 / B 跟 2000 时底池按 7000 均分，B 有机会赢走 A 未被跟注的 3000；现退还后只均分真正争到的 4000。同时消除未跟注残留的单人边池。
- **验证**：不等筹码全押+run-it（A 退还 3000 / 守恒 7000 / B 不超 4000）✅；3 人弃牌盲注→摊牌合并为单一主池 10040 / 守恒 ✅；runit/站起/房主回归全通。

## 🎲 多次发牌 run it N times（2026-07-24，测试服已上）
- **两人 all-in 协商发几次**：亮牌后若**恰两人对局**且还有公共牌未发 → 发起协商——**落后方(equity 低)选发 1~5 次**，**领先方同意**才生效；多人局(3+)固定发 1 次。
- 同意后**底池均分 N 份**，以**已发公共牌为共享底**，从牌堆**依次发 N 组不同 runout**（每组补齐剩余街+烧牌），各自判赢家分得该份（平局再均分该份，余数进第一份）。

## v1 已知限制
- **仅支持双人 heads-up**：双人专属行动顺序
- **采用标准 heads-up 规则**：按钮位 = 小盲(SB)，preflop 先动；非按钮位 = 大盲(BB)，postflop 先动
- **无真正边池**：用"未跟注退还"近似处理 2 人 all-in 不同筹码深度场景，3+ 玩家不同 all-in 金额需要真正的边池系统
- **Bet vs Raise 已区分**：本街无人下注 → Bet（最小 BB）；已有下注 → Raise（须大于当前注）
- **牌局进行中禁止加入**：避免座位索引混乱
- ~~**小盲/大盲写死 10/20**~~：已改为按房间 SNG 级别读取（`gameSB/gameBB`，初始 25/50 升盲表）

## 产品方向（房间类型）
项目目标支持两种房间类型：
- **SNG（Sit-N-Go 单桌锦标赛）**：盲注按时间递增、玩家淘汰制，当前优先级。需配置盲注级别（如 10/20→20/40→…）、涨盲间隔（如 5min）、起始筹码、奖励结构
- **Cash 现金桌（9-max）**：盲注固定，玩家可随时带入/离开。次优先级

服务器房间状态需扩展 `roomType` / `blindLevels` / `currentLevel` / `levelStartTime` 等字段；现有写死的 SMALL_BLIND/BIG_BLIND 应改为从房间配置读取。

### 经济模型（含抽水，后期实现 —— 当前 SNG 是简化版：固定 50 入场、冠军通吃）
- **多人桌 = 现金桌**：2–9 人，盲注固定，随时带入/离开。
- **SNG 锦标赛奖励**：
  - 报名费**可配置**（50 / 100 / 200 / …），不再写死 50
  - 奖池**抽水**：冠军不拿全部。例：2 人共 100 奖池，冠军拿约 85（其余 ~15 为平台抽水/服务费）；多人按奖励结构分配（如前 N 名）
- **现金桌买入/兑换有价差（抽水）**：
  - 买入：例 **110 金币 → 1000 筹码**
  - 离场：例 **1000 筹码 → 100 金币**
  - 即每 1000 筹码进 110、出 100，平台抽 ~10%
- 实现时：`config.buyIn` 可配置；结算时按 `rake` 比例计算冠军/离场金额；现金桌带入/兑出走 `gold↔chips` 汇率（买入价 > 卖出价）。

### 大厅 / 房间创建（参考"德扑之星"商业产品截图）
登录后先进**大厅页**（而非直接进牌桌）：顶部「加入比赛（输入邀请码）」+「创建比赛」两个入口，下方按 `训练赛 / SNG` 分类列出房间卡片（房名、房主、牌型、人数 x/max、级别时间、报名/即将开赛按钮）。

创建房间分两种模式，各自配置项：

**① SNG（单桌锦标赛，含双人 heads-up 升盲）** —— 当前优先
- 牌型：国际扑克 / 短牌 / 奥马哈（先只做国际扑克）
- 比赛牌局名字
- 开赛人数：2–9（2 即 heads-up）
- 报名费 / 记分牌类型（独立记分牌）
- 初始记分牌（起始筹码）：5K–30K
- 级别时间（涨盲间隔）：3–10 min
- 级别结构（盲注递增表）：标准模式
- 奖励分配
- 玩法：盲注按 `级别时间` 递增、淘汰制

**② 训练赛（现金桌 / 好友多人对战）** —— 次优先，多人引擎就绪后做
- 牌型、比赛名字
- 基础分（盲注 SB/BB）：10/20 起，可选到 500
- Ante 设置：0–80
- 级别升级设置（可选开启递增）
- 单次最小带入：2K–8K
- 带入记分牌上限：无限制 / 16K–60K
- 训练时长：0.5–6h
- 单桌最大人数：2–9
- 坐满自动开始：手动 / 2–9 人
- 玩法：盲注固定（除非开启升级），玩家随时带入/离开

> ⚠️ 多人（3–9）需要先补齐**多人游戏引擎**：真正的行动顺序（UTG 起手）、真正边池、按钮轮转。当前 server.js 仅实现双人 heads-up。可先做大厅 + 房间创建 UI 与配置存储，多人对局逻辑随后跟进。多人桌面布局参考 8-max 环形座位截图。
